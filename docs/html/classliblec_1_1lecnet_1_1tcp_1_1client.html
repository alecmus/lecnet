bool liblec::lecnet::tcp::client::connect ( const client_params &  params,
std::string &  error 
)

Connect to a TCP server.

Parameters
paramsServer parameters, as defined in the client_params struct.
Parameters
errorError information.
Returns
Returns true if the client thread was created successfully, else false.

Note that this function returns almost immediately. The actual connection attempt is made on the client thread. To establish the result of the connection attempt wait for connecting to return false and then check the status of connected. Any error encountered in the connection attempt will be written to the error parameter of connected.

◆ connected()

bool liblec::lecnet::tcp::client::connected ( std::string &  error)

Check if the client is connected to a server.

Parameters
errorError information. Note: do not assume this contains any error information just because the client is not connected, e.g. when this function is called immediately after connect but before the actual connection is established (in this case false will be returned but error will be empty). If this function is called immediately after false is returned from connecting and it returns false as well, then error is guaranteed to have some information about why the connection attempt failed. Error information is erased from memory immediately after a call to this function.
Returns
Returns true if client is connected to a server, else false.

It only makes sense to call this function after a successful call to connect and waiting until false is returned from connecting.

Check if the client is currently trying to connect to a server.

Returns
Returns true if the client is in the process of trying to connect to a server, else false.

After true is returned by connect, wait until false is returned by this function, then call connected to establish if the connection was successful.

Get the server response.

Parameters
data_idThe unique ID associated with the data that was sent.
Parameters
receivedThe data received from the server.
Parameters
errorError information.
Returns
Returns true if successful, else false.

A call to send_data_async, and then checking for the status of sending is important for this call to make sense. After a successful call to send_data_async, wait until sending returns false, then call this function. After the response is retrieved using this function it is immediately deleted from memory. A new call to send_data_async can then be used before checking for another response.

◆ running()

bool liblec::lecnet::tcp::client::running ( )

Check if the client thread is running.

Returns
Returns true if the client thread is running, else false.

This function is useful to check if the client is still connected to the server after true is returned from connected. Once false is returned you can be sure the connection has been lost and a call to connect is required to reestablish the connection.
